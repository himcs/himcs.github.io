# The Twelve Factors

# 1 Codebase

使用版本控制系统，如Git管理的代码库。

基于一份代码，可以部署在本地，测试环境，线上环境。

# 2 Dependencies

显示声明依赖，例如Java的Maven，node的npm

# 3 Config

在环境中存储配置,将配置和代码分离

# 4 Backing services

将后端服务当作附加资源。

例如数据库（MySQL)，消息队列（RabbitMQ），缓存系统(Redis)都是后端服务。

系统可以在不改动代码的情况下，通过替换资源地址的方式更换资源。

# 5 Build, release, run

严格分离构建和运行。

  • 构建是将代码转换为可执行包的过程
  • 发布是将可执行包将配置结合
  • 运行是将特定发布包，在执行环境启动

# 6 Processes

以无状态进程运行运行应用。

进程必须无状态且无共享，任何需要持久化的数据都存在后端服务中。

# 7 Port binding

通过端口绑定提供服务。

应用应该自我加载并不依赖服务器容器，即可以绑定任意端口来提供服务，并监听发送到该端口的请求。

# 8 Concurrency

通过进程模型进行扩展

# 9 Disposability

快速启动，优雅终止。

# 10 Dev/prod parity

尽可能的保持开发，预发布，线上环境相同

# 11 Logs

把日志当作事件流。

应用程序不要考虑日志的输出流，不要试图去写或管理日志文件。因该由运行环境捕获，一并发送或归档，例如ELK

# 12 Admin processes

以一次性进程的形式运行admin/management任务
