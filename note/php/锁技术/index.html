由来

由于PHP是没有 Java 语言级别的锁关键字及基础类，单机上可以借助文件锁来实现锁，来完成对并发资源的串行访问。

解决的问题

例如门票超卖问题，一共有 10 张票，不加锁，多个并发访问可能导致票超卖

问题出在，检测票数量与减少票之间不是原子操作导致的。
问题代码，这里使用数据库来存储票的数量

并发访问,会导致票数超卖变成负数

解决方案

使用mysql数据库的事务 for update 排他锁机制

使用事务,在事务中使用 for update 语句锁住这条数据,操作完成后释放.
这种方式局限mysql数据操作,如果有其他的操作依然不能保证原子性.
